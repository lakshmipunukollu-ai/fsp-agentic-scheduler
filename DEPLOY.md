# Deploy FSP Agentic Scheduler

## Railway (recommended)

One Node service serves the API and the built frontend. Postgres is added as a plugin.

### 1. Create a new project

1. Go to [Railway Dashboard](https://railway.app/dashboard) and **New Project**.
2. **Deploy from GitHub repo** and select this repository.
3. Railway will detect the root and use `railway.toml` (build + start commands).

### 2. Add Postgres

1. In the project, click **+ New** → **Database** → **PostgreSQL**.
2. After it’s created, open the Postgres service → **Variables** and copy the `DATABASE_URL` (or use **Connect** to see the connection string).
3. Open your **web service** (the one from the repo) → **Variables** → **Add variable** (or **Reference**):
   - Add **`DATABASE_URL`** and reference it from the Postgres service (e.g. **Variable reference** → select Postgres → `DATABASE_URL`),  
   **or** paste the connection string manually.

### 3. Set environment variables

In the **web service** → **Variables**, ensure:

| Variable        | Value / note |
|-----------------|--------------|
| `NODE_ENV`      | `production` |
| `DATABASE_URL`  | From Postgres (see step 2). |
| `JWT_SECRET`    | A long random string (e.g. generate with `openssl rand -base64 32`). |
| `CORS_ORIGIN`   | Your app URL, e.g. `https://your-app.up.railway.app` (Railway shows it after first deploy). |

`PORT` is set by Railway; no need to add it.

### 4. Deploy

Push to the connected branch (or trigger a deploy from the dashboard). Railway will:

- Run `npm run build` (builds frontend, copies to `backend/public`, builds backend).
- Run `npm run start` (`cd backend && node dist/index.js`).
- Expose the service at a URL like `https://fsp-agentic-scheduler-production.up.railway.app`.

### 5. Migrations and seed (first time)

After the first successful deploy:

1. In the **web service**, open **Settings** → **Deploy** or use the **Shell** (if available).
2. Run migrations and seed from the repo root:

   ```bash
   cd backend && npm run migrate && npm run seed
   ```

   If your shell is already in `backend`, run:

   ```bash
   npm run migrate && npm run seed
   ```

   (You can also run these via a one-off run or a deploy hook; see Railway docs.)

### 6. Open the app

Open the service URL from the Railway dashboard and log in with the seeded users (see backend seed or README for demo accounts).

## Render.com (alternative)

The repo includes a [Render Blueprint](https://render.com/docs/blueprint-spec) (`render.yaml`).

1. **New → Blueprint** in the [Render Dashboard](https://dashboard.render.com), connect this repo.
2. Set **CORS_ORIGIN** on the web service to your app URL (e.g. `https://fsp-agentic-scheduler.onrender.com`). **JWT_SECRET** is generated by the blueprint.
3. After deploy, in the web service **Shell** run: `cd backend && npm run migrate && npm run seed`.

## Local production build

To test a production build locally:

```bash
./scripts/build-for-render.sh
cd backend && PORT=3001 NODE_ENV=production node dist/index.js
```

Then open `http://localhost:3001`. The database must be running (e.g. `docker-compose up -d db`) and migrated/seeded.
